📦 Bàn giao & Quản lý Phiên bản
Giai đoạn 2 của Luồng 3 — Bàn giao toàn diện cho cty thành viên và lưu trữ version
📋 Gói tài liệu bàn giao
BA
Tài liệu nghiệp vụ: Mô tả yêu cầu, Flow xử lý, Acceptance Criteria, Hướng dẫn sử dụng cho user
Dev
Tài liệu kỹ thuật: Solution design, API docs, DB schema thay đổi, Impact analysis, Deployment guide
QA
Tài liệu nghiệm thu: Test case, Kết quả UAT, Biên bản nghiệm thu có chữ ký
PM
Release package: Release Note, Deployment log, Rollback history, Version changelog
🏷️ Quản lý Phiên bản (Version Log)
v
Đặt tên version: v{Major}.{Minor}.{Patch} — Ví dụ v2.1.0 (Major feature), v2.1.1 (Hotfix)
W
Lưu trữ trên Wiki: Mỗi version có 1 trang riêng gồm changelog, tài liệu, link Jira Epic
🔔
Thông báo version mới: Email chính thức gửi cty thành viên kèm Release Note và hướng dẫn upgrade
🗃️
Lưu trữ artifacts: Build artifacts, DB scripts lưu trên repo có tag version tương ứng
🤝 Quy trình Nghiệm thu chính thức
1
Bàn giao môi trường UAT đầy đủ dữ liệu thực tế cho cty thành viên
2
Cty thành viên test trong 3–5 ngày làm việc theo test script được cung cấp
3
Họp nghiệm thu: BoltX + Đại diện cty trình bày kết quả, xử lý feedback
4
Ký biên bản nghiệm thu (cả 2 phía), attach vào Jira Epic
5
CEO BoltX ký Checklist Golive → DevOps deploy Production
📡 Monitor sau Release dự án
1h
1 giờ đầu: Smoke test toàn bộ tính năng chính trên PROD, kiểm tra log
24h
24 giờ đầu: Monitor error rate, response time, DB load — alert ngay nếu bất thường
1w
1 tuần đầu: Daily check dashboard, CX thu thập feedback user, PM tổng hợp issues phát sinh
✅
Sau 1 tuần: PM xác nhận ổn định → Đóng Epic trên Jira → Lưu Version Log final lên Wiki
